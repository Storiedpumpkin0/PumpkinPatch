Vagrant

  Vagrant is a tool for building and managing virtual machine environments in a single workflow (taken from Vagrants website).
  With vagrant one can download pre-packaged boxes and run virtual machines in seconds, as well as run custom boxes. Vagrant works by reading a “Vagrantfile” which contains directions to retrieve, successfully open, provision, and manage virtual machines


  Vagrant requires a file called a “Vagrantfile” in order to run. These files are very simple and only requiring a few things.
 To generate an empty Vagrantfile:
  - Open a shell
  - Navigate to the folder you wish to work out of
  - enter the command: 
 
   vagrant init

 (note: if you know the path/url to a specific box, you can provide it and vagrant will generate a vagrant file for that box)

Installing vagrant

1.) Go to Vagrants website and select the version appropriate for your operating system
2.) A pop up will appear. Click “save file” and wait for the package to download.
3.) Continue through the installer until the installation has completed.

  Using Vagrant

  To run vagrant, open a shell in the folder containing your Vagrantfile and enter the command: 

	vagrant up

  Vagrant will then:
  - Open the box
  - name the vm
  - provision the vm with hardware specifications and environmental variables
  - SSH into the virtual machine
  - Provision the virtul machine with inline commands

  There are many useful vagrant commands that can be ran after the vagrant box has been created. Some examples are:
  - Vagrant halt -- shutdown of the VM
  - Vagrant provision -- forces the provisioning inside the Vagrantfile to run again (useful if it failed the first time)
  - Vagrant ssh -- ssh into the VM
  - Vagrant destory -- completely removes the VM and all of its resources
  - Vagrant -h -- display a complete list of commands
