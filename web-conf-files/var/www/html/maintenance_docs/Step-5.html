Step-5 Configuring the Web Server

This guide will provide instruction on how to build an online repository where the vagrant boxes can be hosted and easily downloaded. This is NOT an in depth apache or http guide, but it will provide enough instruction to get a web server up and running with a few pages and downloadable content.

  Starting Apache

  After apache is installed, we will want to make sure that it can be started. To check the status of the apache service, enter the command: 

	sudo systemctl status httpd

  To start the apache service:

	systemctl start

  And then check the status again to make sure it is running

  Apache can be stopped with:

	systemctl stop httpd

 Then check the status again to be sure it stopped


Writing the web pages

While apache is running it will look in the doccument root for html files.

the typical web root is 

	/var/www/html

For this project there only a few web pages running on apache. We have:

	- an index (the home page)
	- a downloads page
	- Documentation pages.

Index

the index acts as a home page for the service. It contains a brief description of the project and links to other pages.


Downloads

this is where links to the downloadable files will go. In this project there are links to boxes, Vagrantfiles, and iso files. It is important to have the link pointing to the actual path of the file you are linking, and that file needs to have adequate permissions in order to be downloaded

A simple way to ensure files are in directories with correct permissions is to house them in sub-folders within the doccument root.

For this project there are sub-folders for: 

	- boxes 
	- iso_downloads
	- maintenance_docs
	- pictures
	- Vagrant recipies

Documentation

These pages (packer and vagrant on the home page) are general "how to use" pages.

They contain info about how to install and use the tools, as well as useful screenshots.
